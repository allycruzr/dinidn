### 🟡 Supabase Edge Functions workaround (`verify_jwt: false`)
Quando a gente ainda estava usando Belvo, descobri que o gateway de Edge Functions desse projeto estava rejeitando **todos** os JWTs (erro "Invalid JWT") mesmo com sessão válida e publishable key. Workaround foi deployar com `verify_jwt: false` e fazer auth manual chamando `supabase.auth.getUser(jwt)` dentro da função. **Isso vale lembrar** caso um dia a gente volte a usar Edge Functions — pode ser necessário o mesmo workaround. Tanto a causa raiz quanto se é bug do projeto específico ou algo mais amplo ficaram inconclusos.

## Coisas que aprendi / gotchas

1. **Publishable keys novas do Supabase**: formato `sb_publishable_*` em vez de JWT. Devem ser usadas no frontend em vez da legacy anon JWT. Mas não necessariamente resolvem bugs de Edge Functions gateway.

2. **Supabase Edge Functions MCP**: tem `deploy`, `list`, `get`, `get_logs` mas **não tem `delete`** nem gerenciamento de Vault secrets. Cleanup é sempre manual.

3. **OFX 2.x vs 1.x**: OFX 2.x é XML puro (`<CODE>0</CODE>`), OFX 1.x é SGML (`<CODE>0` sem fechamento). Meu parser detecta por `<?xml` no header e aplica conversão SGML→XML só quando necessário. O conversor também é safe pra rodar em XML: usa lookahead pra não adicionar closing tag se já existe.

4. **Formatos de data brasileiros em XLSX**: BTG exporta `10/04/2026 19:35:41` (com hora junto), meu regex inicial exigia `$` no fim → bug. Agora tolera hora. Cuidado com `new Date("10/04/2026")` do JS que interpreta como MM/DD (outubro 4, não abril 10).

5. **Postgres ON CONFLICT**: se duas linhas no mesmo batch de upsert tem a mesma chave única, Postgres rejeita com `21000`. Solução: contador de ocorrências dentro do batch antes de enviar.

6. **OFX "saldo" vs "soma de transações"**: são coisas DIFERENTES. Soma = movimento do período. Saldo = valor atual da conta. OFX traz o saldo em `<LEDGERBAL><BALAMT>`. XLSX geralmente tem em célula separada no header ("Saldo atual: 841,57").

7. **BTG não exporta OFX** — só Excel ou PDF. Por isso o parser XLSX existe.

8. **Belvo Sandbox BR tem 2 bancos fake**: Iron Bank (Advanced login) e OF Mock Bank by Raidiam. Ambos deram erro 500 nos meus testes, então nem pra validar o pipeline serviu. A abordagem correta pra Belvo seria pagar pelo plano Production, que exige aprovação manual (horas a dias).
